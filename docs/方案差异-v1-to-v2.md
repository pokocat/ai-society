# 方案差异记录：v1 → v2

v1 = 仅基于设计仓库初版代码（commit `0fb6c06`）反推的方案（未入库，曾在会话中评审）。
v2 = 基于设计仓库最新 `13a3a3c` + 新增业务基线文档《蜂乐玛社群运营中台-业务逻辑与项目说明》V1.0（SPEC）全面校准后的版本，即同目录《技术方案-v2-工程评审稿.md》。

## 触发差异的输入变化

1. 设计仓库新增 `docs/蜂乐玛社群运营中台-业务逻辑与项目说明.md`（443 行业务基线：边界、对象、流程、规则、状态机、分阶段计划、验收标准）
2. 代码侧变化仅为主题重构：「纯白黄绿黑」（#ffffff / #b6ff00 / #050805）成为默认主题；导航结构与业务组件逻辑无变化

## 未变的决策（会话既定，v2 延续）

- OWL 本体做设计期概念权威（`ontology/bossclub.ttl` + MAPPING.md），不上运行时推理
- 关系链 PostgreSQL 物化路径 `lv1/lv2/lv3_parent`（≤3 级、无环、单推荐人），不上 Neo4j；影响力矩阵 = 层级×身份聚合
- 后端 Java 21 + Spring Boot 3.3 + PostgreSQL 16 + Flyway；契约驱动（openapi → TS client）
- admin-pc 复用设计仓库源码；两小程序 Taro 4 重写（weapp+h5）
- 交付三个子应用：PC 管理后台 / 会员小程序 / 金服小程序
